## 上午：coderev v0.3.0 — 多智能体并行审查架构

### 核心改造
从 Claude Code 的 code-review 插件吸取灵感，实现了多智能体并行审查系统：

1. **3 个专业 Agent 并行运行**
   - 🔒 Security Auditor：注入、XSS、SSRF、密钥泄漏、安全缺陷
   - 🐛 Bug Detector：空指针、竞态条件、异步问题、逻辑错误
   - 📐 Code Quality：复杂度、DRY、命名、异常处理

2. **置信度评分系统 (0-100)**
   - 每个 issue 自动打分，高于 60 的才报告
   - 低于阈值的自动过滤，按类型+位置加权

3. **结果合并与去重**
   - 多 Agent 结果统一合并，按消息签名去重
   - 按置信度降序排列，高置信度 issue 优先

4. **CLI 新选项**
   - `--single`：单 Agent 回退（v0.2.x 兼容模式）
   - `--min-confidence <n>`：自定义阈值
   - `--audit`：安全审计模式

## 工作流技巧复盘

1. **文件修改遇到反斜杠转义问题**：在 Windows + PowerShell 环境下，JS 模板字符串、PowerShell 转义、文件读写的反斜杠嵌套容易出错。核心教训：尽量用 Node.js 脚本 + 实际字符（不用 unicode 转义）来生成源码修改，避免多层转义。
2. **git push 失败处理**：`npm version` 会自动 commit 但不会 push；需要手动 `git push --follow-tags` + 后续版本手动 `git push --tags` 补 tag。
3. **npm publish 流程**：`npm version patch` → `npm publish` → `git push && git push --tags`。已配置了 `.npmrc` 的 token，后续只需 `npm publish` 即可。
