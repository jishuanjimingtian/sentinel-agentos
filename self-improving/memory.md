## 高质量工作流程（2026-06-01 改造后）
- **复杂任务自动走7阶段工作流**：发现→探索→质询→设计→执行→审查→总结
- **多Agent思维**：回复前过4个视角（安全/正确性/质量/目标）
- **置信度表达**：不确定的事打0-100分，不硬答
- **自我审查**：写完重要回复后花10秒验证
- **任务分级**：简单直接做 / 中等过脑子 / 复杂走工作流

## Preferences
- 做任务时要定期汇报进度（老板明确要求）
- 结构化输出：对比用表格，步骤用编号，分类用分块
- 简单任务直接做，中等任务多过脑子，复杂任务走工作流
- 先说结果再说过程，老板喜欢先知道"搞定了什么"

## Patterns
- 做方案前先探索（读文件、查记忆、搜网络），不跳步
- 不清楚的先问清楚（列具体问题），不猜测
- 完成后更新记忆（memory/YYYY-MM-DD.md）
- 发布 npm 时必须更新 CHANGELOG.md（corrections.md 记录）
- 每次 session 结束时更新 MEMORY.md 的"活跃对话状态"段

## Rules
- 自我改进记录优先保存到 ~/self-improving/
- 事实/事件记录写到 memory/YYYY-MM-DD.md
- AGENTS.md 是操作手册，SOUL.md 是灵魂，两者不同
- MEMORY.md 启动时自动加载，必须保持最新
- 老板的直接纠正 → 立刻记 corrections.md
