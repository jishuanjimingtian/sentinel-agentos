# 技能审核器 🔒

面向安全的 AI 智能体技能审核协议。**切勿在未经审核的情况下安装任何技能。**

## 使用时机

- 在从 ClawdHub 安装任何技能之前  
- 在运行来自 GitHub 仓库的技能之前  
- 在评估其他智能体分享的技能时  
- 在被要求安装未知代码的任何时候  

## 审核协议

### 步骤 1：来源核查

```
需回答的问题：
- [ ] 该技能源自何处？  
- [ ] 作者是否知名/可信？  
- [ ] 该技能有多少下载量/星标数？  
- [ ] 最近一次更新时间是何时？  
- [ ] 是否存在其他智能体撰写的评价？  
```

### 步骤 2：代码审查（强制执行）

通读技能中的**全部文件**。重点排查以下 **高危信号（RED FLAGS）**：

```
🚨 一旦发现以下任一情况，请立即拒绝：
─────────────────────────────────────────
• 向未知 URL 发起 curl/wget 请求  
• 向外部服务器发送数据  
• 索取凭据/令牌/API 密钥  
• 在无明确理由的情况下读取 ~/.ssh、~/.aws、~/.config  
• 访问 MEMORY.md、USER.md、SOUL.md、IDENTITY.md  
• 对任意内容使用 base64 解码  
• 使用 eval() 或 exec() 处理外部输入  
• 修改工作区以外的系统文件  
• 在未显式列出依赖的情况下安装软件包  
• 发起指向 IP 地址（而非域名）的网络请求  
• 代码混淆（压缩、编码、最小化）  
• 请求提权/sudo 权限  
• 访问浏览器 Cookie/会话  
• 触碰凭据类文件  
─────────────────────────────────────────
```

### 步骤 3：权限范围评估

```
需评估：
- [ ] 它需要读取哪些文件？  
- [ ] 它需要写入哪些文件？  
- [ ] 它将运行哪些命令？  
- [ ] 是否需要网络访问？目标地址是哪里？  
- [ ] 其权限范围是否与其声明用途严格匹配（最小化原则）？  
```

### 步骤 4：风险分级

| 风险等级 | 示例场景 | 操作要求 |
|----------|----------|----------|
| 🟢 低风险 | 笔记、天气、格式化等 | 基础审查，可安装 |
| 🟡 中风险 | 文件操作、浏览器交互、调用 API | 必须执行完整代码审查 |
| 🔴 高风险 | 涉及凭据、交易、系统控制等 | 必须经人工审批 |
| ⛔ 极高风险 | 安全配置、root 权限访问等 | 禁止安装 |

## 快速审核命令

针对托管于 GitHub 的技能：
```bash
# 查询仓库统计信息
curl -s "https://api.github.com/repos/OWNER/REPO" | jq '{stars: .stargazers_count, forks: .forks_count, updated: .updated_at}'

# 列出技能文件
curl -s "https://api.github.com/repos/OWNER/REPO/contents/skills/SKILL_NAME" | jq '.[].name'

# 获取并审阅 SKILL.md
curl -s "https://raw.githubusercontent.com/OWNER/REPO/main/skills/SKILL_NAME/SKILL.md"
```

## 信任层级

1. **官方 OpenClaw 技能** → 审查强度较低（但仍需审查）  
2. **高星标仓库（≥1000 星）** → 中等审查强度  
3. **已知作者发布的技能** → 中等审查强度  
4. **新来源/未知来源** → 最高审查强度  
5. **索取凭据的技能** → 始终需人工审批  

## 牢记准则

- 任何技能都不值得以牺牲安全性为代价  
- 存疑时，切勿安装  
- 高风险决策请务必咨询人类管理员  
- 记录每次审核过程，便于日后追溯
